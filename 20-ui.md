# User Interface

> *The interface should reveal the ecosystem, not hide it.*

## The interface is not the product

The user interface is the visible expression of the workspace.

It does not define the ecosystem.

It simply helps individuals understand and interact with it.

The interface should always remain secondary to the objects it presents.

---

## Objects before pages

Traditional software is organized into pages.

Dashboards.

Settings.

Forms.

Menus.

The openglacier interface is organized around objects.

Users interact with objects.

Not pages.

---

## Zoom, don't navigate

Navigation should feel natural.

Selecting an object reveals more context.

Not another application.

Not another disconnected page.

Each interaction should preserve orientation and context.

Understanding grows through exploration.

Not navigation.

---

## One interaction model

Every object follows the same interaction principles.

Observe.

Understand.

Act.

Regardless of whether the object represents:

- an individual
- a resource
- an agent
- a mission
- an organization

Users should not need to learn different interfaces for different technologies.

---

## Calm interfaces

Good interfaces do not compete for attention.

Animations explain changes.

Notifications communicate important events.

Colors describe state.

Silence is preferred over noise.

---

## Reality over decoration

Every visual element should communicate meaning.

Status indicators represent real states.

Animations represent real transitions.

Icons represent real entities.

Decoration should never replace information.

---

## Honest interfaces

Disconnected resources remain visible.

Failed missions remain visible.

Unavailable devices remain visible.

The interface should explain reality.

Never hide it.

---

## Progressive complexity

The interface should reveal complexity only when it becomes useful.

Beginners should immediately understand the ecosystem.

Experts should always be able to explore deeper.

Complexity should emerge through interaction.

Never through overload.

---

## Consistency creates confidence

Objects behave consistently.

Actions appear consistently.

States are represented consistently.

Predictability builds confidence.

---

## Explain before automate

People should understand what is happening before being asked to trust automation.

Interfaces should always answer:

What happened?

Why?

What will happen next?

---

## Accessibility by design

The interface should remain usable regardless of:

- age
- technical expertise
- abilities
- devices

Accessibility is not an optional feature.

It is part of good design.

---

## Responsive by nature

The workspace should adapt naturally to different devices.

Large screens provide more context.

Small screens preserve the same concepts.

The interaction model remains identical.

---

## Presence before activity

Objects remain visible even when inactive.

Presence does not depend on activity.

Understanding depends on continuity.

---

## The interface should disappear

The best interface is the one users no longer notice.

People should focus on their ecosystem.

Not on the software itself.
